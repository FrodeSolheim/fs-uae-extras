Muligheder i WHDLoad

  • kan startes via CLI eller Workbench
  • Hukommelses beskyttelse med MMU (hvis tilgængeligt)
  • returnerer helt rent til OS (Afhængig af slaven)
  • Muligt at bruge "disk images" og/eller normale filer
  • variabel chip-hukommelses størrelse for det installerede program fra $2000 op til $200000
  • uendelig fast-memory størrelse for det installerede program
  • intelligent allokering af chip-memory, partiel absolut og partiel uafhængig
  • caching af filer og disk images ved startup og dynamisk i runtime (LRU)
  • automatisk decrunching af filer og disk images ved hjælp af XPK
  • automatisk decrunching af filer hvis Slaven supporterer det (RNC/IMP/TPWM/CRM)
  • emulation af MOVE SR,ea fra User-Mode på 68010+
  • emulation af unsupporteret integer-instructioner på 68060
  • Avanceret debugging support (memory dump, register dump, fil tilgahgs logging, logging af custom register tilgang)
  • Enforcer/Cyberguard og VMM proof så længe optionen NoVBRMove/S ikke bruges

WHDLoad er testet på de følgende maskiner

  • A1200 BlizzardIV 030/50/50 2 MB Chip 16 MB Fast AGA Kick 39.106
  • A2000 GVP 030/25/25 1 MB Chip 1+4 MB Fast ECS Kick 37.175
  • A4000 CS 060/50 2 MB Chip 16 MB Fast CV64 Kick 40.068
  • A4000 CSPPC 060/50 2 MB Chip 144 MB Fast CV64 Kick 40.068
  • x86 Win2k WinUAE 0.8.21r1
  • x86 AmigaXL
  • og mange flere...
