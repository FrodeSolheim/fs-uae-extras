Системные требования

 

...для запуска устанавливаемых программ:

  • Центральный процессор 68000 (рекомендуется процессор не ниже 68010, потому что некоторые slave-модули требуют VBR для функции выхода из программы по нажатию горячей клавиши)
  • Kickstart 2.0 (версия 37)
  • Не менее 1 Мб оперативной памяти (иногда и больше, это зависит от установленной программы)
Эмуляция Кикстарта

Некоторые патчи используют так называемый kickemu для создания полноценной среды AmigaOS для установленной программы. Для работы kickemu требуется образ оригинального кикстарта. Этот образ может быть получен путем считывания кикстарта с реальной Амиги с помощью утилиты GrabKick, которую можно найти на сайтах aminet. (убедитесь, что вы правильно выбрали настройки для ваших дисков (MaxTransfer, Mask), иначе созданный образ может быть испорчен). Поддерживаемая версия образа кикстарта зависит от установленной программы. Информацию о том какой кикстарт должен использоваться, как правило можно найти в прилагаемом к патчу файле ReadMe. Образы кикстартов версий 1.2 и 1.3 могут быть получены путем сохранения их с подходящих A500, кикстарт версии 3.1 может быть сохранён с моделей A500/A600/A2000, A1200 или A4000 который должен быть оборудован ROM Kick3.1. Если образ кикстарта не будет найден программой WHDLoad или образ будет иметь другую версию или не будет полностью идентичен оригинальному, то WHDLoad завершит работу с соответствующим сообщением об ошибке. Также должны присутствовать, соответствующие конкретному образу кикстарта, дополнительные файлы переадресации с расширением .RTB. Эти файлы могут быть найдены на сайтах aminet в архиве util/boot/skick346.lha. Оба файла, образ кикстарта и файл переадресации, должны находиться в директории Devs:Kickstarts/ и обязаны иметь правильные названия. Начиная с версии WHDLoad 17.0, файлы могут также находиться в директории WHDCOMMON:. Следующая таблица показывает правильные названия файлов в зависимости от версии кикстарта:

Начиная с версии WHDLoad 16.5 можно использовать образы кикстартов с компакт-дисков Cloanto Amiga Forever. Образы кикстартов должны быть переименованы в соответствии с таблицей ниже. Файл rom.key должен быть скопирован в Devs:Kickstarts/ или WHDCOMMON:. Некоторые старые патчи требуют kick40068.A1200 и соответсвенно эти патчи не будут работать с образами кикстартов с CD Cloanto Amiga Forever.
Amiga Forever 2008 уже содержит kick40068.A1200 и образы кикстартов автоматически копируются в правильное место.
Amiga Forever 2011 также включает в себя образы kick33180.A500 и kick40063.A600. Для поддержки этих образов требуется WHDLoad версии 17.0.

Опциональные пакеты:

  • Kickstart 3.1 (версия 40), исправляет некоторые ошибки ранних релизов, а также добавляет некоторые новые функции, которые если доступны, используются WHDLoad, например, MemoryPools, повышающий производительность, если в устанавливаемой программе собержится большое количество файлов.
  • для использования данных, которые сжаты по алгоритму XPK, требуется установить XPK-библиотеки, которые можно найти в aminet, любые файлы данных могут быть сжаты с использованием XPK, WHDLoad будет их распаковывать в момент загрузки, распаковка XPK требует дополнительной памяти и поэтому стоит быть внимательным, если используется конфигурация с ограниченным объемом нижней памяти (low memory), файлы, создаваемые WHDLoad, всегда несжаты.
  • для автоматического выбора языка, используемого в некоторых установщиках, используется lowlevel.library, эта библиотека является частью Workbench 3.1, но также может быть установлена на более старых версиях ОС как расширение, если lowlevel.library отсутствует, установщик по-умолчанию будет использовать ангийский язык
  • если установленный WHDLoad будет использовать библиотеку reqtools.library для отображения любых запросов, reqtools должна быть сконфигурирована, к примеру, для того в каком месте будут отображаться запросы, скачать можно с aminet

...для создания новых патчей и программ инсталяции

  • рекомендуется процессор не ниже 68030 с MMU для использования всех возможностей WHDLoad (крутая машина - всегда лучший выбор для разработки программного обеспечения ;-))
  • знание машинных кодов, ассемблера и амижного железа...
  • ассемблер (хорошо, если умеющий оптимизировать)
  • другие программы (hexedit, disasm...)
  • чуть больше оперативной памяти
  • доступ к различным машинам для возможности более широкого тестирования (68000 - 68060, OCS, AGA, GfxCard)
  • аппаратный или программный системный монитор типа AR3, HRTmon, TK
  • кнопка NMI для прерывания выполняющейся программы или входа в отладчик/системный монитор.
