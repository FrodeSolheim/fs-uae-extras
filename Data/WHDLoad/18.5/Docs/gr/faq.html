Συχνές Ερωτήσεις (FAQ)

Ερώτηση: Εντάξει, έχω τον εγκαταστάτη και υπάρχουν προβλήματα είτε με την εγκατάσταση είτε με την εκκίνηση. Είμαι σίγουρος ότι τα έχω κάνει όλα σωστά μέχρι τώρα. Τι κάνω τώρα;
Απάντηση: Τέτοια συμπεριφορά μπορεί να οφείλεται σε διάφορους λόγους:
  • λειψοί/λανθασμένοι τύποι εργαλείων
    Μερικές διορθώσεις δεν θα τρέξουν σωστά (ή και καθόλου) εάν δεν ορίσετε τους σωστούς τύπους εργαλείων (tooltypes) για το σύστημά σας. Διαβάστε τις οδηγίες για να μάθετε ποιοί τύποι εργαλείων υποστηρίζονται. Παραδείγματα από τύπους εργαλείων που μπορεί να βοηθήσουν είναι τα NoCache, NoVBRMove, NoAutoVec, PAL/NTSC, NoMMU.
  • το αρχείο readme που έρχεται με το πακέτο εγκατάστασης
    Αυτό συχνά εξηγεί μερικά συγκεκριμένα προβλήματα, κλειδιά ή τύπους εργαλείων που πρέπει να οριστούν.
  • λάθος έκδοση του WHDLoad
    Οι περισσότερες εγκαταστάσεις χρειάζονται την τελευταία έκδοση του WHDLoad και δεν θα τρέξουν με μια παλιότερη. Ο χρυσός κανόνας είναι να έχετε πάντα την τελευταία έκδοση εγκατεστημένη.
  • το μηχάνημά σας δεν είναι συμβατό
    Ειδικά ένας 68040/060 CPU ή μερικοί επιταχυντές/άλλο hardware μπορεί να δημιουργήσουν προβλήματα. Δοκιμάστε να απενεργοποιήσετε όλο το επιπλέον hardware και δοκιμάστε να ξεκινήσετε την εγκατάσταση και πάλι.
  • μη υποστηριζόμενη έκδοση
    Πολλά εμπορικά παιχνίδια έχουν πουληθεί σε περισσότερες από μια εκδόσεις (έκδοση πλήρης τιμής/οικονομική επανέκδοση/έκδοση περιοδικού, έκδοση PAL/NTSC κτλ). Φυσικά μια εγκατάσταση μπορεί να λειτουργήσει μόνο αν έχει γραφτεί ειδικά για μια έκδοση του παιχνιδιού. Εάν έχετε διαφορετική έκδοση του ίδιου παιχνιδιού, δεν θα δουλέψει. Σε αυτή την περίπτωση, ο μόνος τρόπος να χρησιμοποιήσετε την εγκατάσταση είναι να επικοινωνήσετε με τον δημιουργό της εγκατάστασης και να του ζητήσετε να υποστηρίξει την έκδοσή σας. Στις περισσότερες περιπτώσεις ο δημιουργός θα σας βοηθήσει και θα διορθώσει την έκδοσή σας, μέσα σε μερικές ημέρες.
  • παλιότερη έκδοση της εγκατάστασης
    Όταν εντοπιστεί και αφαιρεθεί ένα σφάλμα, η νέα έκδοση της διόρθωσης είναι διαθέσιμη στην ιστοσελίδα του WHDLoad. Πάντα προμηθευτείτε την τελευταία έκδοση της εγκατάστασης. Παρακαλώ σημειώστε ότι η τελευταία έκδοση δεν είναι πάντα αυτή στο Aminet. Μερικές διορθώσεις δεν κυκλοφορούν καθόλου στο Aminet, και πολλές κυκλοφορούν με μερικές εβδομάδες καθυστέρησης!
  • ανακαλύψατε ένα νέο σφάλμα
    Παρόλο που όλες οι διορθώσεις δοκιμάζονται πριν την κυκλοφορία τους, ένα σφάλμα ίσως να διαφύγει χωρίς να ανακαλυφθεί. Αν νομίζετε ότι έχετε ανακαλύψει ένα σφάλμα, γράψτε μια λεπτομερής αναφορά του προβλήματός σας στον δημιουργό της διόρθωσης. Για αυτό, χρησιμοποιήστε την φόρμα αναφοράς σφαλμάτων. Προσπαθήστε να αναφέρετε το πρόβλημά σας όσο πιο ακριβές γίνεται. Αν δεν ενημερωθούμε για τα σφάλματα δεν μπορούμε να τα διορθώσουμε. Σημείωση: Η ΜΟΝΗ σωστή τοποθεσία για αναφορές προβλημάτων/σφαλμάτων είναι η φόρμα αναφοράς σφαλμάτων και όχι το comp.sys.amiga.games ή κάποιο άλλο forum, διότι δεν τα διαβάζουν όλοι οι προγραμματιστές.
Ερώτηση: Έχω ένα παιχνίδι αλλά δεν μπορώ να βρω εγκατάσταση για αυτό.
Απάντηση: Κάντε μια λίστα των επιθυμιών σας. Μπορείτε επίσης να συμπεριλάβετε παιχνίδια που είναι ήδη εγκαταστήσιμα στον σκληρό δίσκο από μόνα τους (αρχεία που αντιγράφονται και/ή εγκατάσταση που παρέχεται μαζί με το ίδιο το παιχνίδι) που έχουν προβλήματα (ακόμα και με απλά προβλήματα υποβάθμισης ή χωρίς έξοδο). Θυμηθείτε ότι ένα άτομο μπορεί να διορθώσει περίπου 25 παιχνίδια το χρόνο μόνο. Σημειώστε με προτεραιότητες ποιά είναι τα αγαπημένα σας (π.χ. ! και +) και ποιά όχι (-), και ελέγξτε ποιά παιχνίδια έχουν ήδη εγκαταστάσεις δίσκου.
Ερώτηση: Είναι δυνατόν να χρησιμοποιήσω το WHDLoad για την εγκατάσταση ενός demo/παιχνίδι που χρησιμοποιεί το λειτουργικό σύστημα;
Απάντηση: Γενικώς δεν είναι δυνατό για ένα demo/παιχνίδι που τρέχει μέσα στο WHDLoad να έχει πρόσβαση σε πράγματα του OS από έξω, από όπου έχει ξεκινήσει το WHDLoad. Εσείς ως συγγραφέας του αναγκαίου προγράμματος Slave πρέπει να διορθώσετε όλες τις προσβάσεις στο OS. Είτε απενεργοποιείτε τέτοιες προσβάσεις είτε πρέπει να τις εξομοιώσετε κατάλληλα. Για την εξομοίωση υπάρχουν τα αποκαλούμενα πακέτα KickEmu. Είναι διαθέσιμα για διαφορετικές εκδόσεις Kickstart (π.χ. 'Sources/whdload/kick13.s'). Χρησιμοποιώντας ένα είδωλο ενός πραγματικού kickstart, δημιουργούν ένα πλήρες περιβάλλον OS μέσα στη μνήμη του WHDLoad. Υπάρχει ένα επιπλέον αρχείο ReadMe σχετικά: 'Sources/whdload/kick.readme'.
