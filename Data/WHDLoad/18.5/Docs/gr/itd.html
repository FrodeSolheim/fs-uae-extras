ITD * Image To Disk

Εργαλείο για εγγραφή των disk images πίσω σε συσκευές όπως δισκέτες. Το ITD περιέχεται μόνο στο πακέτο DEV, όχι στο πακέτο USR.
Αντιγράψτε το στο C: ή κάπου αντίστοιχα

Επιλογές

File/A,Device,Format/S,BS=BlockSize/N,ASync/S,ForceOverwriteLargeDevice/S
File/A όνομα του disk image που θα εγγραφεί στη συσκευή
Device η συσκευή στην οποία το ITD θα γράψει το image
προεπιλογή "DF0:"
παράδειγμα "RAD:"
Format/S εάν ενεργοποιηθεί ο δίσκος θα φορμαριστεί καθώς εγγράφεται, για τις περισσότερες συσκευές αυτό δεν έχει διαφορά
BlockSize/N η ποσότητα των δεδομένων που διαβάζεται και γράφεται κατά κάθε κύκλο, η προεπιλογή είναι το μέγεθος ενός track (BlocksPerTrack*SizeBlock)
εάν χρησιμοποιείτε μεγάλες συσκευές ίσως βελτιώσετε τη απόδοση χρησιμοποιώντας μεγαλύτερο μέγεθος block αλλά έχετε υπόψη ότι μερικές συσκευές δεν λειτουργούν σωστά με μεγέθη block μεγαλύτερα από 130,560 ($1fe00) ή ακόμα και 65,024 ($fe00) +
+ASync/S + +if enabled ITD will write asynchron to the destination device
+don't use if source file and destination device are located on the same +physical device, because it will slow down the operation
+if at least one of source file and destination device supports DMA it will +significant improve the performance of the copy operation +
+ForceOverwriteLargeDevice/S Κανονικά το ITD δεν θα διαβάσει από αρχεία και δεν θα γράψει σε συσκευές που είναι μεγαλύτερες από 2.000.000 bytes. Χρησιμοποιώντας αυτή την επιλογή θα λειτουργήσει σε αρχεία και συσκευές κάθε μεγέθους.

Επιστρεφόμενος κωδικός

Είναι 0 για επιτυχία διαφορετικά είναι 20.

Ιστορικό

Το ITD μπορεί να γράψει σε κάθε συσκευή σαν το trackdisk. Για την αποφυγή προβλημάτων (π.χ. ζημιά στο σκληρό σας δίσκο) δεν γράφει σε συσκευές μεγαλύτερες από 2.000.000 bytes εκτός αν χρησιμοποιηθεί η επιλογή ForceOverwriteLargeDevice/S. Εάν η λειτουργία εγγραφής πάει έξω από το όριο των 4 GiB θα χρησιμοποιήσει εντολές TrackDisk64.
ΔΕΝ θα υπάρξει διαδικασία επαλήθευσης.
Αν το image είναι μικρότερο από τη συσκευή θα πάρετε ένα προειδοποιητικό μήνυμα. Το ITD δεν θα αλλάξει τα υπόλοιπα tracks σε αυτή τη περίπτωση αλλά θα στρογγυλέψει το μέγεθος σε ένα πλήρες block και θα καθαρίσει τον υπολειπόμενο κενό χώρο. Αν το image είναι μεγαλύτερο θα πάρετε επίσης ένα μήνυμα και το ITD θα γράψει όσα δεδομένα μπορεί να χωρέσει στη συσκευή.
