Benodigdheden

Minimale systeemeisen om installs te draaien:

  • CPU 68000 (68010+ is aanbevolen omdat sommige installs de VBR nodig heeft voor de 'stop via een toets' kenmerk)
  • Kickstart 2.0 (versie 37)
  • Minimum van 1MB aan geheugen (soms meer, het ligt aan het geïnstalleerde programma)
Kickstart Emulatie

Sommige installaties gebruiken een zogeheten kickemu om een complete AmigaOS omgeving te creëren voor het geïnstalleerde programma. Voor deze kickemu is een originele kickstart image nodig. Deze image kan gehaald worden uit een echte Amiga, met programma's zoals GrabKick of iets vergelijkbaars welke gevonden kan worden op aminet (weet zeker dat er een correcte setup voor de diskettes zijn). (MaxTransfer, Mask) of anders kan het gebeuren dat het gemaakte image bestand corrupt is. De ondersteunde image versie hangt af van een install. U kunt de informatie vinden welke image gebruikt moet worden in het ReadMe bestand. De image van kickstart versies 1.2 en 1.3 kan alleen gehaald worden vanuit een geschikte A500, voor versie 3.1 kan het gehaald worden vanuit een A500/A600/A2000, A1200 of A4000 welke uitgerust is met een Kick3.1 ROM. Als de image niet gevonden kan worden door WHDLoad of als de image de verkeerde versie heeft of veranderd is, zal WHDLoad stoppen met een geschikte foutmelding. Tevens moet er een relocatiebestand met de extensie .RTB aanwezig zijn die gelijk is aan de kickstart image. Dit relocatiebestand kan gevonden worden op aminet in het archief util/boot/skick346.lha. Beide bestanden, de kickstart image en de relocatiebestand moeten in de directory Devs:Kickstarts/ staan en de juiste naam hebben. Bij het opstarten van WHDLoad met de versie 17.0 kunnen de bestanden ook in de directory WHDCOMMON: gezet worden. De tabellen hier beneden bevatten de namen van de bestanden met de bijbehorende kickstart versies. Vanaf WHDLoad versie 16.5 kunnen ook de kickstart images van de Cloanto Amiga Forever CD's gebruikt worden. De images moeten hernoemt worden volgens het tabel onderaan. Tevens zal de rom.key gekopieerd moeten worden naar Devs:Kickstarts/ of WHDCOMMON:. Sommige oudere installaties hebben de kick40068.A1200 image nodig, deze installaties werken niet met de Cloantos ROMs.
Amiga Forever 2008 bevat ook de kick40068.A1200 en de images worden automatisch gekopieerd naar de juiste plek.
Amiga Forever 2011 bevat ook kick33180.A500 en kick40063A600 rom bestand. Deze rom bestanden hebben WHDLoad versie 17.0 nodig.

optie pakketten:

  • Kickstart 3.1 (versie 40), deze versie verhelpt een aantal problemen van eerdere uitgaves, het heeft ook een aantal nieuwe functies, welke gebruikt worden door WHDLoad wanneer beschikbaar, zoals MemoryPools. Deze zorgt voor betere prestaties wanneer veel bestanden gebruikt worden door een install.
  • om XPK-compressie data bestanden te kunnen gebruiken, moet de XPK-libraries geïnstalleerd zijn. Deze kunnen gevonden worden op aminet, alle data bestanden kunnen gecomprimeerd worden met het gebruik van XPK, WHDLoad zal deze decomprimeren tijdens het laden, XPK decompressie heeft extra geheugen nodig en er moet hiermee rekening gehouden worden voor systemen met weinig geheugen, bestanden geschreven door WHDLoad zijn altijd ongecomprimeerd.
  • voor automatische taal selectie in sommige installs, word de lowlevel.library gebruikt, deze library in onderdeel van Workbench 3.1 maar kan ook geinstalleerd worden bij vorige uitgaves van besturingssystemen als een uitbreiding. Als de lowlevel.library niet aanwezig is dan zal standaard voor een install de taal Engels zijn.
  • als geïnstalleerd, zal WHDLoad de reqtools.library gebruiken om elke requestervensters te kunnen openen, reqtools kunnen geconfigureerd worden, bijvoorbeeld waar de requester zal verschijnen, het kan gedownload worden van href="http://aminet.net/package/util/libs/ReqToolsUsr.lha">aminet

Systeem aanbevelingen om nieuwe Slaves te schrijven of programma's te installeren

  • Aanbevolen is een 68030+ met MMU om alle kenmerken van WHDLoad te gebruiken (een krachtige machine is altijd goed om software te ontwikkelen ;-))
  • Machinecode, assembler, Amiga hardware kennen ...
  • Een assembler (moet kunnen optimaliseren)
  • Een paar andere programma's zijn handig (hexedit, disasm, ...)
  • Een aantal megabytes aan geheugen
  • Toegang tot verschillende machines om te kunnen testen (68000 - 68060, OCS, AGA, GfxCard)
  • Een hardware of software freezer zoals AR3, HRTmon, TK
  • Een NMI knop om een lopend programma te pauzeren of om een software-monitor/freezer te betreden
